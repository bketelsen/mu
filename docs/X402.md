# x402

Mu uses x402 as an outbound payment client. The authenticated owner or owner
agent can call a configured remote x402-enabled service, receive its `402`
payment requirement, pay in the configured asset, and retry that remote request.

## Configuration

```bash
export X402_PAY_TO="0xYourOwnerWallet"
export X402_FACILITATOR_URL="https://api.cdp.coinbase.com/platform/v2/x402"
export X402_NETWORK="eip155:8453"
export X402_ASSETS="USDC,EURC"
```

Use `mu x402` to inspect configuration without exposing secrets. Set owner spend
limits before enabling agent-initiated payments.

## Authentication boundary

Mu does not offer inbound x402 payment access. Requests to Mu's web, API, MCP,
or A2A surfaces still require owner authentication. An incoming payment proof is
never a replacement for a session, PAT, OAuth credential, CLI token, or MCP
credential.

## Implementation

`wallet/x402client.go` handles owner-initiated remote calls and
`wallet/spendlimit.go` enforces spending limits. x402 configuration affects only
the outbound client and does not change route access control.
